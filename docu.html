I. INTRODUCTION

OVERVIEW OF THE PROJECT

The Wash Connect Laundry Management System is a desktop-based software application designed to automate and modernize the daily operations of a local laundry business. It serves as a digital replacement for traditional manual logbooks, aiming to streamline transaction recording, price calculation, and customer record management. By integrating a user-friendly interface with a secure cloud-based database (Supabase), the system ensures accurate data handling and provides business owners with real-time insights into their operations.

a. PROBLEM STATEMENT

The project addresses the following issues in the current manual system:

  1. Calculation Errors: Frequent mistakes in manually computing prices for excess weight and add-ons.
  2. Inefficient Data Retrieval: Time-consuming search for customer history in physical logbooks.
  3. Lack of Sales Monitoring: Difficulty in generating immediate daily or monthly income reports.
  4. Data Security Risks: Physical records are vulnerable to loss or damage.

b. OBJECTIVES OF THE STUDY

The goal is to develop the Wash Connect System with the following capabilities:

  1. To develop a Secure Login System for admins.
  2. To implement an Automated Pricing Module (calculates based on 6kg min, excess weight, add-ons).
  3. To create a Customer Management Module for storing and searching client details.
  4. To provide a Real-time Sales Dashboard.
  5. To utilize a Cloud Database for data backup.

c. SCOPE AND LIMITATIONS

Scope: The system covers User Authentication, Order Management, Status Tracking, Sales Reporting, and Customer Database.

Limitations: The system requires an internet connection for cloud syncing and is designed for desktop use only (no mobile app).

II. PROJECT BACKGROUND

a. DESCRIPTION OF THE PROJECT DOMAIN

The project targets the laundry service industry, specifically Small-to-Medium Enterprises (SMEs). These businesses often rely on manual labor for record-keeping. This system introduces automation to improve efficiency, accuracy, and data security in this service sector.

b. CURRENT SYSTEM OR PROCESS

The existing manual process involves:

  1. Recording: Writing orders in a logbook.
  2. Calculation: Using a calculator for totals.
  3. Tracking: Flipping through pages to check status.

Limitations: Slow, error-prone, and lacks data security.

c. JUSTIFICATION OF THE PROPOSED SYSTEM

The new system improves upon the current process by automating calculations (eliminating errors), enabling instant search (saving time), and backing up data to the cloud (preventing loss).
